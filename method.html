Method

How Suede Audit calculates a readiness score.

Every audit fetches one public page — the homepage you submit — plus three discovery files already public on that domain: robots.txt, llms.txt, and sitemap.xml. Suede Audit does not log in, does not crawl beyond those four documents, and does not run prompts inside ChatGPT, Perplexity, Gemini, or any other answer engine.

How the score is weighted

Each check carries a severity: high, medium, or low. High-severity checks count five times toward the score, medium-severity checks count three times, and low-severity checks count once. The readiness score is the percentage of weighted checks a site passes, rounded to the nearest whole number.

  • 90 and above — A
  • 80–89 — B
  • 70–79 — C
  • 55–69 — D
  • Below 55 — F

Three pillars

Every check rolls up into one of three pillars, each scored the same way as the overall report:

  • Discoverability — reachability, crawl policy, canonical routing, and the public discovery files (robots.txt, llms.txt, sitemap.xml).
  • Entity evidence — structured identity data: JSON-LD presence, organization type, and verified identity links.
  • Content clarity — page meaning, supporting content, share metadata, and response weight.

What the score does not prove

A high score is not proof of an AI citation. Buyer-prompt evidence requires separate, timestamped runs inside each answer engine, which this audit does not run. The score measures whether an answer engine can technically reach and parse a page — not whether it will recommend, rank, or cite it.

Limits and caching

Each browser gets one free audit. The finished report is cached in that browser's local storage, so its report link can reopen without using another scan. Suede Audit accepts public HTTP and HTTPS URLs only; it will not inspect private, internal, or non-routable addresses.
